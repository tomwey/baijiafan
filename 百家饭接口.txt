Host http://120.132.57.133/api/v1

备注：每个接口调用会返回message，建议可以使用message的内容进行有好的接口返回提示

1.获取验证码
POST /auth_codes/fetch
参数  mobile 手机号，必须

2.用户登录
POST /account/login
参数  mobile 手机号，必须
     code 短信验证码，必须
     
3.修改用户个人资料
POST /user/update_profile
参数  token 登录成功后，会返回Token, 必须
     nickname 用户名，可选参数
     avatar 图片数据，可选参数
     signature 个性签名，可选参数

4.获取用户个人资料
GET /user/me?token=用户Token

5.获取用户发布的菜
GET /user/items?token=用户token

6.发布菜单
POST /items/create
参数     token Token，必须
        title 标题，必须
        price 价格，整数，必须
        quantity 数量，整数，必须
        expired_at 截止日期，字符串，必须，格式为：2000-01-01 12:00:00
        address 地址，字符串，必须
        latitude 纬度，数字符串，必须
        longitude 经度，数字符串，必须
        service_modes 服务方式，字符串，必须
        image 图片数据，二进制数据，可选
        note 温馨提示，可选，字符串
        
7.查看某个菜单详情
GET /items/show/:id  id为条目id

8.根据当前位置获取菜单列表
GET /items/list
参数     latitude 纬度，数字符串，必须
        longitude 经度，数字符串，必须
        range 覆盖范围，以米为单位，整数，可选，备注：如果range参数不传值，表示列表模式，否则为地图模式
        
9.点赞与取消点赞
POST /item/:method 其中method参数的值固定为like或unlike
 参数 item_id 为某个菜条目的id，必须

注意：客户端自己处理点赞的状态保留

10.修改发布的菜品
POST /item/update
参数     token Token，必须
        item_id 条目id, 必须
        title 标题，可选
        price 价格，整数，可选
        quantity 数量，整数，可选
        expired_at 截止日期，字符串，可选，格式为：2000-01-01 12:00:00
        address 地址，字符串，可选
        latitude 纬度，数字符串，可选
        longitude 经度，数字符串，可选
        service_modes 服务方式，字符串，可选
        image 图片数据，二进制数据，可选
        note 温馨提示，可选，字符串
        
11.删除一个条目
POST /item/delete
参数   token  Token, 必须
      item_id 条目id, 必须
      
12.下订单
POST /orders/create
参数  
   token, 用户Token, 必须
   item_id, 条目id, 必须
   service_modes, 服务方式, 必须
   address, 服务地址, 必须
   quantity, 份数, 必须
   fee, 总价, 可选
   note, 备注, 可选
 
13.获取我的订单列表
GET /orders/list
参数
  token, 用户Token, 必须
  type, 列表数据类型，1表示我吃过的订单，2表示我卖过的订单, 必须
  page, 当前页码, 可选，默认为1
  size, 分页大小, 可选，默认为30

例如：/orders/list?token=xxxxx&type=1 或者 /orders/list?token=xxxxx&type=2&page=2

14.获取卖家未接受的订单
GET /orders/unaccepted
参数 
  token, 用户Token, 必须
例如：/orders/unaccepted?token=xxxxxx

15.卖家接受订单
POST /order/accept
参数
  token, 用户Token, 必须
  order_no, 订单号, 必须

16.取消订单
POST /order/cancel
参数
  token, 用户Token, 必须
  role, 操作角色，值为buyer(买家，普通用户)或seller(卖家，发布者)之一, 必须
  order_no, 订单号, 必须
  
17.卖家完成订单
POST /order/complete
参数
  token, 用户Token, 必须
  order_no, 订单号, 必须
